# 서가의 봉인, 아카이브 루미나 (Lumina Archive)

찢긴 봉인서의 페이지를 복원해 잃어버린 이름을 되살리는 수집·가챠형 모바일 웹 게임.

## 무엇인가

`bat1120/ai-game-collection` 저장소에 담긴 단일 HTML 게임입니다. 커밋 메시지(`deploy: AI Game Studio 자동 제작 게임`)대로 AI 에이전트가 자율적으로 제작한 작품으로, 별도 빌드 과정이나 외부 라이브러리 없이 `index.html` 하나에 데이터·로직·렌더링·사운드가 모두 들어 있습니다.

세계관은 "복원 사서"가 된 플레이어가 봉인서에서 떨어져 나간 캐릭터들의 인장을 모아 기록실로 불러들이는 이야기입니다. 게임 시작 시 비주얼노벨 형식의 프롤로그(PROLOGUE · 찢어진 첫 페이지)가 재생되며, 첫 동료 "세린 아르카"가 합류한 뒤 4단계 튜토리얼이 이어집니다.

## 게임 루프와 조작

핵심 루프는 화면 상단에 노출되는 **소환 › 도감 › 편성 › 전투 › (반복)** 흐름을 따릅니다.

1. **소환(가챠)** — 결정(💎)이나 티켓(🎟️)으로 캐릭터를 뽑습니다. 1회/10회/티켓 10회 소환을 지원하며, 카드 공개 연출과 결과 요약 화면이 캔버스로 그려집니다.
2. **도감** — 획득 캐릭터의 일러스트·프로필·서사·스탯·스킬을 열람하고 보유율을 확인합니다. 미획득 캐릭터는 잠금(🔒) 표시됩니다.
3. **편성** — 최대 3명으로 팀을 구성합니다. 탱커·딜러·힐러가 모두 있으면 균형 편성 보너스가 붙습니다.
4. **전투** — 편성한 팀으로 3웨이브(마지막 웨이브는 보스) 자동 전투를 진행합니다. 승리 시 결정·인장 조각·경험치를 받고 다음 스테이지가 해금됩니다.

조작은 터치/클릭 기반입니다. 비주얼노벨은 탭(또는 Space/Enter)으로 진행하고 "건너뛰기"가 가능하며, 소환 연출도 탭으로 넘깁니다. 전투에는 배속(x1~x3)과 포기 버튼이 있습니다.

### 자원과 성장

- **💎 결정** — 소환 재화. 신규 계정은 3,200 결정과 티켓 5장으로 시작합니다.
- **🎟️ 티켓** — 10회 소환용 재화.
- **📜 인장 조각** — 중복 캐릭터를 환원하거나 전투 보상으로 얻으며, 캐릭터 레벨업에 사용합니다.
- 캐릭터 레벨업은 레벨당 8% 스탯 상승, 사서(계정) 레벨은 전투 경험치로 오릅니다.

### 가챠 확률

- ★5 0.7% · ★4 8.0% · ★3 91.3%
- 90회 천장: 누적 90회째에 ★5 확정 (★5 획득 시 천장 진행도 초기화)
- 중복 캐릭터는 등급별 인장 조각(★5 50 / ★4 20 / ★3 8)으로 환원

## 캐릭터

총 6명이 데이터로 정의되어 있습니다. 각 캐릭터는 이름·칭호·등급·역할·소속·속성(빛/어둠/대지/바람/번개)·성격·서사·대사·스킬·기본 스탯을 가집니다.

- 세린 아르카 (★5, 서포터/리더, 빛)
- 루나 벨잉크 (★5, 광역 딜러, 어둠)
- 노아 렌즈 (★5, 암살자/디버퍼, 바람)
- 미르카 북쉴드 (★4, 탱커, 대지)
- 엘리사 아스트라 (★4, 힐러, 빛)
- 하린 모듈 (★3, 유틸리티, 번개)

캐릭터 일러스트는 `assets/unit0.png`~`unit5.png`를 사용하며, 이미지 로드에 실패하면 캔버스로 그린 대체 초상화가 표시됩니다.

## 전투 동작

자동 전투로, 각 액터는 쿨다운에 따라 행동합니다. 힐러는 체력이 낮은 아군을 회복하고, 광역 딜러는 적 전체를 타격하며, 그 외는 단일 대상을 공격합니다. 크리티컬(20% 확률, 1.6배), 피격 점멸·넉백, 히트스톱, 화면 흔들림, 파티클·데미지 팝업 등의 연출이 캔버스에 렌더링됩니다. 적은 일반 "훼손된 페이지"와 보스 "잔향의 봉인수"로 나뉘며, 스테이지가 오를수록 체력·공격력이 증가합니다.

## 기술 스택

- 순수 HTML / CSS / JavaScript (프레임워크·외부 의존성 없음)
- Canvas 2D — 가챠 공개 연출, 전투 렌더링, 대체 초상화, 비주얼노벨 파티클 효과
- Web Audio API — 클릭음, 등급별 소환 효과음, 전투 타격음을 코드로 합성
- `localStorage` — 계정 목록과 세이브 데이터 저장 (`lumina_accounts_v1`, `lumina_save_v1_<이름>`)
- GitHub Pages 배포

## 플레이 방법

브라우저에서 GitHub Pages 주소로 접속하면 바로 플레이할 수 있습니다.

https://bat1120.github.io/ai-game-collection/

또는 저장소의 `index.html`을 내려받아 브라우저에서 직접 열어도 동작합니다. 진행 상황은 브라우저 `localStorage`에 저장됩니다.

- 첫 화면에서 사서 이름을 입력해 계정을 생성하거나, "게스트로 시작"으로 바로 플레이할 수 있습니다. 기존 계정은 목록에서 이어하기가 가능합니다.
- URL 뒤에 `?test=1`을 붙이면 QA용 게스트 계정으로 즉시 시작합니다.

## 핵심 파일 구조

```
ai-game-collection/
├─ index.html        # 게임 전체 (데이터·로직·UI·렌더링·사운드)
└─ assets/
   ├─ unit0.png      # 세린 아르카
   ├─ unit1.png      # 루나 벨잉크
   ├─ unit2.png      # 미르카 북쉴드
   ├─ unit3.png      # 엘리사 아스트라
   ├─ unit4.png      # 노아 렌즈
   └─ unit5.png      # 하린 모듈
```

`index.html` 내부는 주석으로 구획이 나뉘어 있습니다: DATA(캐릭터·프롤로그 정의), STATE/세이브, ASSETS(이미지·대체 초상화), SCENES(화면 전환), 로그인, 오프닝 VN, 튜토리얼, 홈, 가챠, 리빌 연출, 도감, 상세, 편성, 전투, 와이어링, 초기화.
